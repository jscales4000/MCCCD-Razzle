**What:** Defines the contract-ready signal names used by this CH5-Svelte touchpanel. Names here match the UI bindings in the Svelte app and are intended to be imported into the SIMPL or SIMPL# processor project. Join numbers are not included — those are allocated during Contract Editor build.

**Why:** The CH5 UI needs a stable namespace before the processor program is built so the front end and backend stay aligned. Documenting the signals reduces guesswork during manual SIMPL wiring and makes the UI behavior visible in SIMPL Debug once the contract is built and imported.

**How:** Use the `<ROOM_NAME>` symbol as the contract namespace in Contract Editor. Start from `contracts/<ROOM_NAME>.cce`, review or rename placeholders, then build the contract in Contract Editor so `.cse2j`, `.chd`, or `.g.cs` outputs are generated by the official toolchain. Import those generated files into SIMPL Windows or SIMPL# Pro — never hand-author them.
